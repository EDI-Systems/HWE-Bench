Please design a programmable video interface emulator that acts as an intermediary bridge between a media source and a display device. The core architecture should include a transparent physical passthrough for high-speed multimedia streams, an intercepted low-speed communication bus for device identification, a non-volatile memory block to store customized display profiles, a central processing unit for data management, an external communication interface for user configuration, and a dual-input power arbitration network that seamlessly switches between the power provided by the video cable and an external auxiliary power source.

The operational workflow relies on separating the video payload from the hardware handshake process. Once the power arbitration module establishes a stable supply for the logic circuits, the system allows high-speed video signals to flow through untouched. However, when the media source queries the display's capabilities, the low-speed bus routes this request to the onboard memory block, which replies with the spoofed identification data. Simultaneously, the central processing unit listens to the external communication interface; if a user uploads a new display profile, the processor rewrites the memory block and electronically toggles the connection detection signal. This forces the media source to re-initiate the handshake protocol and read the newly updated parameters without any physical cables being unplugged.
